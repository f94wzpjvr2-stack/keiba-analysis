# 導入・使用マニュアル

## A. 最初の30分で行うこと

### A-1. GitHubへコードを置く

1. GitHubへログインします。
2. 新しいリポジトリを作成します。
3. リポジトリ名を `keiba-hybrid-system` にします。
4. この配布フォルダの中身をすべてアップロードします。
5. mainブランチに保存します。

コードには個人的なレース履歴を入れません。`data/live/` は `.gitignore` によってGitHub管理の対象外です。

### A-2. Codexへリポジトリを接続する

1. Codexを開きます。
2. GitHubアカウントを接続します。
3. `keiba-hybrid-system` を選択します。
4. 最初のタスクとして、テストを実行させます。

Codexへの初回指示：

```text
AGENTS.mdを最初に読み、このリポジトリの構成を確認してください。
python -m pip install -e ".[dev]" と python -m pytest -q を実行し、
失敗があれば修正してください。仕様を勝手に変更せず、
変更したファイル、変更理由、テスト結果を最後に報告してください。
```

### A-3. Colabを開く

GitHubの `notebooks/keiba_colab.ipynb` を開き、「Open in Colab」から起動します。

設定セルの `GITHUB_REPO` を自分のリポジトリ名に変更します。

## B. Colabの役割

Colabは開発場所ではなく、レース当日の実行画面です。

Colabで行うこと：

- Google Driveの接続
- 最新コードの取得
- 馬の採点入力
- オッズテキストの貼り付け
- EV計算
- 資金配分
- CSV保存
- 回収率確認

Colabで行わないこと：

- 大規模なコード修正
- 配点変更の直接書き換え
- テストなしの関数追加
- 個人データのGitHub保存

## C. ChatGPTから受け取る入力形式

毎レース、ChatGPTへ次の形式で出力させます。

```csv
horse_no,horse_name,ability,suitability,pace,training,paddock,win_prob_input,win_odds
1,馬名A,32,15,14,7,8,,5.8
2,馬名B,29,16,15,8,7,,9.4
```

`win_prob_input` は、ChatGPTが明示的な勝率を出した場合だけ小数で入力します。例：18%なら `0.18`。空欄なら採点から暫定推定します。

## D. オッズの貼り付け

### ワイド

```text
1-2 5.2-6.1
1-3 8.4-9.2
```

下限オッズを使って保守的に計算します。

### 馬連

```text
1-2 12.4
1-3 20.8
```

### 三連複

```text
1 1-2-3 18.5
2 1-2-4 22.1
```

人気順位が先頭にあっても解析できます。

## E. 結果の読み方

### ability_rank
採点値による能力順位です。オッズは反映しません。

### model_prob
モデルが推定した的中確率です。暫定値であり、真の確率ではありません。

### ev
`model_prob × odds` です。

- 1.00：理論上の損益分岐
- 1.03以上：初期設定の購入候補
- 高すぎるEV：確率推定または入力ミスを疑う

### stake
クォーターケリーを基礎にした保守的な購入額です。相関する馬券を多数買う危険があるため、券数と1点あたりの上限を設定しています。

## G. アルゴリズムを変更してよい条件

原則として次を満たしてから変更します。

- 同じ問題が複数レースで再現
- 変更前後を別期間で比較可能
- 購入前に利用できる情報だけを使用
- ROIだけでなくサンプル数と最大ドローダウンも確認
- 旧バージョンの結果を残す

## H. エラー時の切り分け

### `ModuleNotFoundError: keiba_ev`

Colabのインストールセルを再実行します。GitHubリポジトリ名が正しいか確認します。続くセルで `PROJECT_DIR/src` が `sys.path` に追加されているか確認します。

### Git cloneに失敗

リポジトリがprivateの場合は認証が必要です。最初はコードだけをpublicリポジトリにし、個人データをDriveへ分離する運用が簡単です。

### オッズが0件になる

貼り付け形式が想定と違います。該当する数行をCodexへ渡し、パーサーのテスト追加と修正を依頼します。

### EVが異常に高い

馬番の対応、単勝オッズ、勝率入力、小数と百分率の混同を確認します。
