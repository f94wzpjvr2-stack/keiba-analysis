# Codexへ送る実用プロンプト集

## 1. 初回点検

```text
AGENTS.mdを読んでから作業してください。
リポジトリを点検し、python -m pip install -e ".[dev]" と
python -m pytest -q を実行してください。
エラーがあれば、仕様を変えずに最小修正してください。
最後に、変更ファイル、変更理由、テスト結果、残る制約を報告してください。
```

## 2. バグ修正

```text
次のエラーを再現して修正してください。

[エラー全文を貼る]

要件：
- 最初に失敗するテストを追加する
- 原因を特定して最小修正する
- 既存CSV形式を壊さない
- python -m pytest -q を実行する
- 修正内容を日本語で要約する
```

## 3. 新しいオッズ形式への対応

```text
odds_parser.pyに、次の実データ形式を読み取る機能を追加してください。

[オッズの数行を貼る]

要件：
- 全角・半角ハイフンとタブ・空白を許容
- 馬番を昇順のselectionへ正規化
- 重複行は最後の値を採用
- 異常行は黙って誤読せず、解析件数を確認できるようにする
- テストを追加し、全テストを実行
```

## 4. 集計機能の追加

```text
results.csvとbets.csvを使い、競馬場、距離帯、馬場、券種、アルゴリズム版別に
購入額、払戻、利益、ROI、件数を集計する機能を追加してください。
サンプル数が少ない行には警告列を付けてください。
READMEとColabノートブックも更新し、テストを追加してください。
```

## 5. 配点変更の検証

```text
配点を直接上書きしないでください。
configに新バージョンを追加し、旧版を残したまま比較できるようにしてください。
学習期間と検証期間を分離し、データ漏洩がないことをテストしてください。
比較指標はROI、購入数、的中率、利益、最大ドローダウンです。
```

## 6. PRレビュー

```text
この変更をレビューしてください。
特に、結果情報の先読み、確率の過大評価、予算超過、100円単位違反、
CSV互換性、テスト不足を確認してください。
重大な問題だけを優先順位付きで報告してください。
```
